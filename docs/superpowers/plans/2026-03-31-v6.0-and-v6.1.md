### Task 7: Deepen /ship skill (129 → 200 LOC)

Add to `plugin/skills/ship/SKILL.md`:

**a) CI monitoring with error handling** (expand Phase 3):
```markdown
### CI monitoring

After creating PR:
```bash
# Watch CI checks
gh pr checks [PR-number] --watch

# If a check fails:
gh run view [run-id] --log-failed
```

**If CI fails:**
1. Read the failure log: `gh run view --log-failed`
2. Diagnose: is it a test failure, lint error, build error, or infra issue?
3. Fix locally, push, wait for re-run
4. Do NOT merge with failing CI

**Common CI issues:**
- Missing env vars → check repository secrets
- Wrong Node/Python version → check CI config vs local
- Test timeout → check if test needs longer timeout or has a real bug
- Lint failure → run linter locally first: `npm run lint`
```

**b) Post-deploy verification** (after Phase 3):
```markdown
### Post-deploy verification

After deployment completes:

1. **Health check** — hit the health endpoint if one exists:
   ```bash
   curl -sf https://[deployed-url]/health || echo "UNHEALTHY"
   ```

2. **Smoke test** — verify the core feature works:
   - Can the main page load?
   - Does the API respond to a basic request?
   - Are logs showing normal traffic?

3. **Monitor for 2 minutes** — watch for error spikes:
   ```bash
   # If using Cloudflare:
   wrangler tail --format=pretty 2>/dev/null | head -20
   ```

4. **If something's wrong** — rollback:
   ```bash
   # Revert the merge commit
   git revert -m 1 HEAD
   git push origin main
   # Redeploy previous version
   ```
```
