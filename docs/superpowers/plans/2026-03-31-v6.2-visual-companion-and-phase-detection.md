# v6.2: Visual Companion + Phase Detection Implementation Plan

> **For agentic workers:** Execute tasks in order. Each task is independently committable.

**Goal:** Add a browser-based visual companion for interactive design sessions (mockups, diagrams, clickable options) and phase-aware context injection that detects where you are in the workflow and injects only the next step.

**Architecture:** The visual companion is a Node.js HTTP + WebSocket server that watches a content directory for HTML files, serves them with a themed frame template, and captures user clicks via WebSocket events. Phase detection uses git state + file existence checks to determine the current workflow phase. Both integrate into existing skills (/design uses the visual companion, inject-prompt-context uses phase detection).

**Tech Stack:** Node.js (HTTP + WebSocket, no external deps), HTML/CSS, Bash

### Task 1: Create the visual companion server

**Files:**
- Create: `plugin/scripts/visual-companion/server.cjs`

- [ ] **Step 1: Create the server**

A Node.js HTTP + WebSocket server with zero external dependencies. Key capabilities:
- Serve HTML files from a content directory
- Watch for new files and broadcast reload via WebSocket
- Wrap content fragments in frame template (full documents served as-is)
- Capture click events via WebSocket, write to events file (JSONL)
- Idle timeout (30 min), owner process tracking, random port selection

The server must:
- Implement WebSocket RFC 6455 frame encoding/decoding (no ws library)
- Use `fs.watch()` for content directory monitoring
- Auto-detect full document vs fragment (check for `<!doctype` or `<html>`)
- Output startup JSON to stdout: `{type, port, url, screen_dir, state_dir}`
- Write `server-info` to state_dir for agent reconnection
- Check owner PID periodically (60s), shutdown if parent exits
- Clear events file when new screen detected

Target: ~350 lines. Use CommonJS (`.cjs`) for compatibility.

```bash
#!/usr/bin/env bash
# Start the visual companion server
# Usage: start-server.sh --project-dir /path/to/project [--host 0.0.0.0] [--url-host localhost] [--foreground]
set -euo pipefail

SCRIPT_DIR="$(cd "$(dirname "$0")" && pwd)"

# Parse arguments
PROJECT_DIR=""
HOST="127.0.0.1"
URL_HOST=""
FOREGROUND=0

while [ $# -gt 0 ]; do
  case "$1" in
    --project-dir) PROJECT_DIR="$2"; shift 2 ;;
    --host) HOST="$2"; shift 2 ;;
    --url-host) URL_HOST="$2"; shift 2 ;;
    --foreground) FOREGROUND=1; shift ;;
    *) shift ;;
  esac
done

[ -z "$URL_HOST" ] && URL_HOST="$HOST"

# Create session directory
SESSION_ID="$$-$(date +%s)"
if [ -n "$PROJECT_DIR" ]; then
  SESSION_DIR="$PROJECT_DIR/.claudetools/design/$SESSION_ID"
else
  SESSION_DIR="/tmp/claudetools-design-$SESSION_ID"
fi

CONTENT_DIR="$SESSION_DIR/content"
STATE_DIR="$SESSION_DIR/state"
mkdir -p "$CONTENT_DIR" "$STATE_DIR"

# Auto-detect foreground mode
[[ "${MSYSTEM:-}${MSYS:-}${CODEX_CI:-}" != "" ]] && FOREGROUND=1

export BRAINSTORM_DIR="$SESSION_DIR"
export BRAINSTORM_HOST="$HOST"
export BRAINSTORM_URL_HOST="$URL_HOST"
export BRAINSTORM_OWNER_PID="$PPID"

if [ "$FOREGROUND" -eq 1 ]; then
  node "$SCRIPT_DIR/server.cjs"
else
  nohup node "$SCRIPT_DIR/server.cjs" > "$STATE_DIR/server.log" 2>&1 &
  # Wait for server to start and output info
  for i in $(seq 1 30); do
    [ -f "$STATE_DIR/server-info" ] && break
    sleep 0.1
  done
  [ -f "$STATE_DIR/server-info" ] && cat "$STATE_DIR/server-info"
fi
```

- [ ] **Step 2: Create stop-server.sh**

```bash
#!/usr/bin/env bash
# Stop the visual companion server
# Usage: stop-server.sh SESSION_DIR
set -euo pipefail

SESSION_DIR="${1:?Usage: stop-server.sh SESSION_DIR}"
PID_FILE="$SESSION_DIR/state/server.pid"

if [ -f "$PID_FILE" ]; then
  PID=$(cat "$PID_FILE")
  kill "$PID" 2>/dev/null || true
  sleep 2
  kill -9 "$PID" 2>/dev/null || true
  echo '{"type":"server-stopped","reason":"explicit-stop"}' > "$SESSION_DIR/state/server-stopped"
fi

# Clean up temp sessions
case "$SESSION_DIR" in
  /tmp/*) rm -rf "$SESSION_DIR" ;;
esac
```

### Task 4: Create phase detection library

**Files:**
- Create: `plugin/scripts/lib/phase-detect.sh`

- [ ] **Step 1: Create the library**

```bash
#!/usr/bin/env bash
# phase-detect.sh — Detect current workflow phase from session state
#
# Returns one of: discover, design, plan, build, test, review, ship, done, unknown

detect_phase() {
  local cwd="${1:-.}"

  # Not a git repo → unknown
  git -C "$cwd" rev-parse --is-inside-work-tree >/dev/null 2>&1 || { echo "unknown"; return 0; }

  local branch
  branch=$(git -C "$cwd" rev-parse --abbrev-ref HEAD 2>/dev/null || echo "")

  # On main/master → done or not started
  case "$branch" in
    main|master) echo "done"; return 0 ;;
  esac

  # Check for plan files
  local has_plan=0
  [ -d "$cwd/docs/plans" ] && [ -n "$(ls "$cwd/docs/plans/"*.md 2>/dev/null)" ] && has_plan=1

  # Check for commits since branch point
  local commit_count
  commit_count=$(git -C "$cwd" rev-list --count main..HEAD 2>/dev/null || echo "0")

  # Check for review evidence
  local has_review=0
  git -C "$cwd" log --oneline -20 2>/dev/null | grep -qi 'review\|reviewed\|code review' && has_review=1

  # Check for PR
  local has_pr=0
  command -v gh &>/dev/null && gh pr view HEAD --json state 2>/dev/null | grep -q '"state"' && has_pr=1

  # Decision tree
  if [ "$has_pr" -eq 1 ]; then
    echo "ship"
  elif [ "$has_review" -eq 1 ]; then
    echo "ship"
  elif [ "$commit_count" -gt 0 ] && [ "$has_plan" -eq 1 ]; then
    echo "review"
  elif [ "$has_plan" -eq 1 ] && [ "$commit_count" -eq 0 ]; then
    echo "build"
  elif [ "$has_plan" -eq 1 ]; then
    echo "build"
  else
    echo "design"
  fi
}

# format_phase_context PHASE → echoes phase-specific guidance
format_phase_context() {
  local phase="${1:-}"
  case "$phase" in
    design) echo "[phase:design] No plan exists yet. Start with /design to explore and plan." ;;
    build)  echo "[phase:build] Plan exists. Execute it with /build (test-first for each task)." ;;
    review) echo "[phase:review] Code committed. Run /review for 4-pass code quality check." ;;
    ship)   echo "[phase:ship] Review done. Run /ship to create PR, verify CI, and merge." ;;
    done)   echo "[phase:done] On main branch. Start new work on a feature branch." ;;
    *)      echo "" ;;
  esac
}
```
