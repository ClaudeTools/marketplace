# Documentation Quality Checklist

Standards based on Jekyll, Hugo, Docusaurus, and GitHub Docs conventions.

## Frontmatter Schema

### Required Fields

Every documentation file must have these fields:

| Field | Format | Example | Validation |
|-------|--------|---------|------------|
| `title` | Descriptive string | `Authentication Flow` | Not empty, not generic (Untitled, Document, Draft, TODO) |
| `description` | One-line summary, 10+ chars | `How OAuth2 works end to end` | Not empty, min 10 characters |
| `updated` | ISO date YYYY-MM-DD | `2026-04-08` | Valid date, not in future, within 90 days |

### Recommended Fields

These fields improve discoverability, governance, and maintenance:

| Field | Format | Values | Purpose |
|-------|--------|--------|---------|
| `status` | Enum | `draft` `active` `review` `deprecated` | Lifecycle tracking — deprecated docs get archived |
| `author` | String | `Jane Smith` | Maintenance ownership — who to ask about this doc |
| `type` | Enum | `guide` `reference` `decision` `tutorial` `overview` `changelog` `api` `runbook` | Classification — enables filtered views and type-specific templates |
| `tags` | YAML list | `[auth, oauth, security]` | Categorization — enables search and cross-referencing |

### Optional Fields

| Field | Format | Purpose |
|-------|--------|---------|
| `slug` | URL-safe string | Custom URL path independent of filename |
| `keywords` | YAML list | SEO/search optimization terms |
| `audience` | String | Target reader (e.g., "backend developers", "new hires") |

### Complete Example

```yaml
---
title: Authentication Flow
description: How OAuth2 authorization works for API clients
updated: 2026-04-08
status: active
type: reference
author: Jane Smith
tags: [auth, oauth, api, security]
---
```

### Minimal Valid Example

```yaml
---
title: Deployment Checklist
description: Steps to deploy the application to production
updated: 2026-04-08
---
```

## Document Type Guide

Choose the right type for each document:

| Type | Use For | Typical Structure |
|------|---------|-------------------|
| `guide` | How-to instructions, walkthroughs | Steps, prerequisites, outcomes |
| `reference` | API docs, config options, schemas | Tables, parameter lists, examples |
| `decision` | Architecture Decision Records (ADRs) | Context, options, decision, consequences |
| `tutorial` | Learning-oriented, step-by-step | Building toward a complete working example |
| `overview` | High-level explanations, concepts | Diagrams, relationships, context |
| `changelog` | Version history, release notes | Date-ordered entries |
| `api` | Endpoint documentation | Methods, parameters, responses, errors |
| `runbook` | Operational procedures, incident response | Steps, escalation, rollback |

## Status Lifecycle

```
draft → active → review → active (refreshed)
                       ↘ deprecated → archived
```

- **draft** — Work in progress, not yet reliable
- **active** — Current and maintained
- **review** — Flagged for accuracy check (manually or by staleness threshold)
- **deprecated** — Superseded or no longer relevant; will be archived

## File Naming

- [ ] Filename is kebab-case: `setup-guide.md` not `Setup Guide.md`
- [ ] Filename reflects content: `auth-flow.md` not `notes.md`
- [ ] No spaces, underscores, or uppercase in filenames

## Content Standards

- [ ] Top-level heading (H1) matches the `title` frontmatter field
- [ ] Overview section explains what the doc covers and why it exists
- [ ] No empty sections or placeholder text (`[TODO]`, `TBD`, etc.)
- [ ] Code blocks specify language for syntax highlighting
- [ ] Internal links use relative paths (`./other-doc.md` not absolute)
- [ ] External links are to stable URLs (docs, not blog posts)

## Directory Structure

- [ ] Each docs/ subdirectory has an `index.md`
- [ ] Each subdirectory has a `_template.md` showing expected frontmatter
- [ ] Deprecated docs have `status: deprecated` (not silently abandoned)
- [ ] `docs/archive/` directory exists for archived documents
- [ ] Index files are auto-generated by `/docs-manager reindex`
